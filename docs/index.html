GitHub Token Setup

GitCTF requires a GitHub Personal Access Token (PAT) to interact with repositories. Your token is stored locally in your browser only and never sent to any server.

1

GitHub PAT 생성하기

GitHub Settings → Developer Settings → Personal Access Tokens → Tokens (classic)

직접 이동하기 →

필요 권한: repo, notifications

2

토큰 저장

3

CLI에서 사용하기

# 환경변수로 설정 (권장)
export GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"

# 또는 명령어마다 --token 옵션 사용
python3 gitctf.py hash --token "ghp_xxx..." --conf config.json
4

Token 검증

Quick Start Guide

Prerequisites

  • Python 3.11+
  • Docker
  • Git
  • GPG (optional, for encrypted submissions)
  • GitHub account with PAT token

Installation

git clone https://github.com/hy30nq/gitCTF.git
cd gitCTF
pip install -r requirements.txt
export GITHUB_TOKEN="your_token_here"

참가자 워크플로우

1

서비스 확인 — 상대팀 서비스가 정상 동작하는지 확인

python3 gitctf.py verify-service --team alpha --branch main \
  --conf config.json
2

익스플로잇 작성 — 서비스의 취약점을 공격하는 Docker 기반 익스플로잇 작성

# templates/skeletons/exploit/ 를 복사해서 시작
cp -r templates/skeletons/exploit/ my-exploit/
# exploit.py 수정 후 테스트
3

익스플로잇 검증 — 로컬에서 자동 검증 실행

python3 gitctf.py verify-exploit --exploit ./my-exploit \
  --service-dir ./target-service --branch bug1 --conf config.json
4

제출 — 검증 통과 시 GitHub Issue로 제출

python3 gitctf.py submit --exploit ./my-exploit \
  --service-dir ./target-service --branch bug1 \
  --target alpha --conf config.json

강사 워크플로우

# 환경 초기 설정
python3 gitctf.py setup --conf config.json

# 커밋 해시 확인
python3 gitctf.py hash --conf config.json

# 자동 채점 시작
python3 gitctf.py eval --conf config.json

# 점수 확인
python3 gitctf.py score --conf config.json

Git 튜토리얼

# 저장소 복제
git clone https://github.com/ORG/your-team-repo.git

# 브랜치 생성 및 이동
git checkout -b bug1

# 변경사항 커밋
git add .
git commit -m "your message"
git push origin bug1

Docker 튜토리얼

# 이미지 빌드
docker build -t my-service .

# 컨테이너 실행
docker run -d -p 4000:4000 my-service

# 컨테이너 확인
docker ps

# 컨테이너 종료
docker rm -f $(docker ps -q)
